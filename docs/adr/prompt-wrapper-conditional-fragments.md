# Prompt-wrapper conditional fragments (Option A) ŌĆö the in-band seam for the `promptGuidance` nudge namespace

**Context.** The `promptGuidance.testFirst` nudge (prd `prompt-guidance-test-first`) needs to STRENGTHEN one mid-block sentence of the worker-prompt wrapper that lives in `CLAIM-PROTOCOL.md` (the canonical text the runner reads verbatim, never a TS literal). The wrapper extractor in `prompt.ts` (`extractCanonicalWrapperTemplate`) takes the first fenced block under the "prompt handed to the work agent" heading VERBATIM, so the strengthened text must be expressible from the markdown + a resolved boolean ŌĆö not from TS prose. Three seam shapes were considered (per the task's Open question):

- **A ŌĆö single wrapper, conditional fragment delimited inline.** Markers `<!-- if promptGuidance.testFirst --> ŌĆ” <!-- else --> ŌĆ” <!-- /if -->` carry both branches inside ONE wrapper block; a small post-extraction pass strips markers + the unselected branch based on the resolved flag.
- **B ŌĆö two named wrapper variants under sub-headings.** Two full wrapper blocks ("default" / "test-first") selected by the resolved flag.
- **C ŌĆö append-only line.** A line appended IFF the flag is on.

**Decision.** **Option A**, with HTML-comment markers and an `<!-- else -->` branch carrying the historic soft text so the OFF path is BYTE-IDENTICAL to today. The strengthened text REPLACES (does not append to) the soft "TDD where the task asks for it" sentence when the nudge is on. The post-extraction transform (`applyPromptGuidance` in `prompt.ts`) runs AFTER the verbatim extractor and BEFORE `<slug>`/`<prd>` substitution; the extractor stays a pure verbatim extractor.

**Why.** B duplicates the entire ~70-line wrapper to vary one line ŌĆö re-introducing the same byte-identical-mirror burden this repo already pays between `skills/setup/protocol/` and `work/protocol/`, now squared, and adds a second heading scan to the extractor for no extractor simplification. C only reads cleanly at the END of the block, but the soft sentence is MID-block (load-bearing prose above + below), so an append point would orphan the strengthened line from the sentence it strengthens. A is the smallest TRUE delta, keeps one source of truth at the natural position, and makes the OFF-path byte-identity guard structural (it is just `applyPromptGuidance(template, {testFirst: false}) === <pre-marker template>`). The HTML-comment markers render invisibly so the protocol doc still reads cleanly to a human.

**Consequences.** The marker convention is now an extraction-contract that any future `promptGuidance` member (e.g. `preferSmallDiffs`) reuses ŌĆö same `<!-- if promptGuidance.<member> -->ŌĆ”<!-- else -->ŌĆ”<!-- /if -->` shape, one regex pass handles every fragment, members are independently resolvable. The `verify` gate's semantics are untouched ŌĆö this is a NUDGE, never a gate. The two protocol-doc copies (`skills/setup/protocol/CLAIM-PROTOCOL.md` SOURCE + `work/protocol/CLAIM-PROTOCOL.md` MIRROR) stay byte-identical per the repo's "edit the SOURCE" rule.
